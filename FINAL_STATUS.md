## 🎯 Your Next 6 Steps (6 minutes total)

### 1️⃣ Create GitHub Account (2 minutes)
```
Go to: https://github.com/join
Sign up with email (free forever)
```

### 2️⃣ Create Vercel Account (1 minute)
```
Go to: https://vercel.com/signup
Sign up with GitHub (simplest option)
```

### 3️⃣ Push Your Code to GitHub (2 minutes)
```powershell
cd "d:\foundation website"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/restored-kings-foundation.git
git push -u origin main
```
Replace `YOUR_USERNAME` with your actual GitHub username.

### 4️⃣ Deploy to Vercel (1 minute)
```
Go to: https://vercel.com/new
Select: "Import GitHub Repository"
Paste URL: https://github.com/YOUR_USERNAME/restored-kings-foundation
Click: "Import Project"

Settings:
- Framework: (blank)
- Build Command: (leave empty)
- Output Directory: public

Click: "Deploy"
```

### 5️⃣ Visit Your Live Site ✓
Vercel gives you a URL like:
```
https://restored-kings-foundation-xxxx.vercel.app
```

## 📞 Common Questions

**Q: Is the site secure?**
A: Yes! Vercel provides free HTTPS, input validation is built-in, no sensitive data in code.

**Q: Will forms work?**
A: Contact/Volunteer/Newsletter forms show success states. To actually save submissions, you'll need to add a backend (see BACKEND_SETUP.md).

**Q: How do donations work?**
A: The Stripe payment form is ready. You'll need to add your Stripe API keys and create a backend endpoint.

**Q: Can I update content easily?**
A: Yes! Edit HTML files, push to GitHub, and Vercel auto-deploys in <1 minute.

**Q: Who can see my code?**
A: Your GitHub repo is public (good for transparency), but you can make it private anytime.

**Q: What if I want a custom domain?**
A: Very easy! Follow the "Add Custom Domain" section in VERCEL_DEPLOY.md.

## 📋 Quick Troubleshooting

| Problem | Solution |
|---------|----------|
| URL shows 404 | Check `vercel.json` output directory = `public` |
| Styling broken | Check CSS path is `/css/styles.css` |
| Images missing | Add to `public/images/` folder |
| Pages not updating | Wait 1 minute after push to GitHub |
| Can't push to GitHub | Check git remote: `git remote -v` |
